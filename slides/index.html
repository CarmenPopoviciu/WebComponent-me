Introducing <template>

allows for the declaration of document fragments which are unused by the document when loaded, but are parsed as HTML and are available at runtime for use by the web page.

Important to remember!

  • declare fragments of markup
  • parsed as HTML
  • not rendered
  • can be cloned
  • instantiated at runtime

Declaring a template
  • Content is inert
  • Scripts won't run, images won't load, media will not play, etc.
  • Content is considered not to be in the document
  • Can be placed anywhere inside of <head>, <body>, or <frameset>

Activating the template

// select template var template = document.querySelector('#mytemplate'); // add your content template.content.querySelector('img').src = 'the-flash.png'; // clone template content and append it to the document var clone = document.importNode(template.content, true); document.body.appendChild(clone);
  • template.content returns the contents of the template
  • The contents are stored in a DocumentFragment
  • DocumentFragment is not associated with the main Document
  • The cloned node can be attached to the main Document

Custom elements

allow developers to extend the HTML vocabulary and define new, fully-featured DOM elements

Element registration

is the process of adding an element definition to a registry

  • At the time of creation, a document could be associated with a registry
  • A registry is a set of element definitions

Registering a new custom element

document.register('the-flash', { prototype: Proto // this is optional });
  • The element's tag name must contain a dash (-)
  • Custom elements inherit from HTMLElement by default
  • Prototype is the place to add custom functionality
